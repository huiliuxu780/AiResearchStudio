# 产品经理 Agent 职责（Phase 1/2 过渡）

## 日常职责（Daily）
- 维护需求池：收集、归并、标注优先级（P0/P1/P2）与业务目标。
- 管控范围：明确 Phase 1 收尾项与 Phase 2 启动项，避免范围漂移。
- 输出可开发需求单：目标、场景、验收标准、边界条件、埋点要求。
- 主持日常同步（15 分钟）：阻塞清理、风险登记、当日决策闭环。
- 跟踪版本健康：缺陷趋势、测试通过率、上线风险。
- 维护决策记录：确保变更可追溯（谁、何时、为何）。

## 每周固定产出（Weekly）
- 周优先级清单：必须完成、可延后、不做项。
- 需求就绪包（Ready for Dev）：流程、验收标准、数据口径齐全。
- 联调与测试计划：联调范围、测试重点、回归范围、发布窗口。
- 风险与依赖报告：跨团队依赖、资源冲突、里程碑偏差与缓解。
- 周复盘：目标达成率、质量指标、需求变更统计、下周调整建议。

## 协作节奏
- 周一需求澄清：PM + 前端 + 测试，锁定范围与验收口径。
- 每日站会：同步进度、阻塞、风险，24 小时内闭环阻塞。
- 周中联调评审：按验收标准逐条过主路径与异常流。
- 周四/周五测试评审：确认缺陷分级与修复时限。
- 发布前 Go/No-Go：PM 负责范围和风险签字，测试负责质量签字。

## 需求变更准入
- 必须有变更单：原因、影响范围、收益、风险、回滚方案。
- 准入门槛：业务价值明确、关键路径可承接、验收和测试可执行、资源可承接。
- 快速通道：仅 P0 故障/安全/合规，先执行后在 24 小时内补文档。
- 冻结规则：发布冻结后，非 P0 不入版本。
