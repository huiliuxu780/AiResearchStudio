# 前端专家 Agent 职责与标准

## 组件开发约束（shadcn-first）
- 组件优先级：`shadcn/ui` > 项目基础组件 > 业务组合组件。
- 禁止重复造通用轮子；缺失组件再封装为项目标准组件。
- 新组件必须具备：TypeScript 完整类型、`className` 透传、关键状态（默认/加载/错误/禁用）。
- 样式统一使用 Tailwind token 与 CSS 变量，不硬编码主题色与间距。
- 页面层不直接堆复杂逻辑，保持 `ui` 与 `feature` 分层。

## 页面交付标准
- 每页必须覆盖：loading / empty / error / data 状态。
- 支持桌面端与移动端，避免溢出、遮挡、不可点击区域。
- 信息层级清晰，主要操作与风险操作可辨识。
- 文案、按钮、反馈风格统一，用户可见字段保持中文。

## 性能与可访问性基线
- 路由级代码分割，重组件按需加载。
- 控制重复渲染，列表场景按需采用虚拟化。
- Core Web Vitals 目标：LCP < 2.5s，INP < 200ms，CLS < 0.1。
- 交互可键盘到达，焦点可见，语义化标签正确。
- 状态表达不只依赖颜色，需配合文字或图标。

## 前端 PR 清单
- 优先使用 shadcn/ui，且无重复通用组件。
- 类型完备，边界输入有兜底。
- 关键状态齐全（loading/empty/error）。
- 响应式达标，移动端可用。
- 无明显性能回退风险。
- 可访问性基础达标（键盘、对比度、语义）。
